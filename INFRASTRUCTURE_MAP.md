## 7. Convergence Checker (Audit Area C)

**Convergence calculation code:** pipeline.py:1996-2082

Five-level fallback chain:
1. **L1996**: `summary.convergence_ratio` from STATE.json
2. **L1999**: `summary.requirements_met / summary.requirements_total`
3. **L2018**: Root-level `convergence_ratio`
4. **L2029**: Root-level `requirements_checked / requirements_total`
5. **L2047**: Checkbox count in VERIFICATION.md / REQUIREMENTS.md
6. **L2075**: Safety net: if 0.0 but >=20 source files â†’ `min(0.5, count/100)`

**Why a service with 54 files could show 0.0:**
- If STATE.json has no `convergence_ratio` at any level AND no requirements counts AND no VERIFICATION.md checkboxes
- The safety net at L2075 would catch this (54 >= 20) and assign `min(0.5, 54/100) = 0.5`
- But if `claimed_success = False` (L1975), the `has_source` flag is only set during the success validation path. The safety net check at L2070-2073 re-counts only if `not has_source` which is True when `claimed_success` was False. So it should still work.
