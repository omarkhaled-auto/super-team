**Description:** `generate_init_sql()` at compose_generator.py:670-678 has dual-name view generation but the CREATE VIEW statements are commented out (prefixed with `--`). This means they won't execute.

**Impact:** If builder code uses different table naming conventions (e.g., `journalentries` vs `journal_entries`), queries fail. However, since the builder gets explicit schema instructions in CLAUDE.md, this is unlikely to be an issue in practice.

**Location:** compose_generator.py:675-677

### GAP-3: Fix Loop Uses `agent_team` Not `agent_team_v15` (LOW)

**Description:** `fix_loop.py:119` prefers `agent_team_v15` but falls back to `agent_team`. However, it does NOT use `_filtered_env()` from pipeline.py -- it has its own simpler env filtering at line 111 that strips `ANTHROPIC_API_KEY` and `OPENAI_API_KEY`, whereas the pipeline's `_filtered_env()` intentionally KEEPS those keys.

**Impact:** Fix pass builder subprocess may not have the API key it needs. However, the main fix path goes through `run_fix_pass()` in pipeline.py:3312-3624 which uses `ContractFixLoop.feed_violations_to_builder()`, and that function handles its own env setup.

**Mitigation:** The fix loop is only used for the contract-based fix mechanism; the primary fix path uses the pipeline's own builder subprocess management.
